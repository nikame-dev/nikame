## 🚀 Project Overview

gen-ai-studio is a high-performance backend generated by [NIKAME](https://github.com/nikame-framework/nikame). 

### Selected Modules

- **postgres**: PostgreSQL relational database with optional replication and pooling

- **minio**: MinIO S3-compatible object storage

- **prometheus**: Prometheus metrics collection and alerting

- **langfuse**: LangFuse LLM observability and tracing

- **prefect**: Prefect dataflow and ML pipeline orchestration

- **mlflow**: MLflow experiment tracking and model registry

- **vllm**: High-efficiency LLM serving with PagedAttention

- **github_actions**: GitHub Actions — Production-ready CI/CD pipelines

- **otel_collector**: OpenTelemetry Collector for vendor-agnostic telemetry ingestion

- **tempo**: Grafana Tempo high-scale distributed tracing backend

- **loki**: Grafana Loki log aggregation system

- **grafana**: Grafana dashboards with auto-provisioned datasources

- **qdrant**: Qdrant vector database for AI/ML


### Active Features

- **rag-pipeline**

- **semantic-search**

- **tracing**

## 👷 Development Workflow

### Adding a New Endpoint
1. Open `app/main.py`.
2. Add a new route handler using `@app.get` or `@app.post`.

### Database Migrations (Alembic)
If you change your SQLAlchemy models:
```bash
nikame exec api alembic revision --autogenerate -m "description"
nikame exec api alembic upgrade head
```

### Background Jobs (Celery)
If background jobs are enabled, add new tasks in `app/worker.py` and call them with `.delay()`.

## 🕵️ Troubleshooting

| Issue | Potential Cause | Fix |
|-------|-----------------|-----|

| Database connection refused | Service down or config error | `Run `nikame up` and check if the `postgres` container is healthy.` |

| Authentication failed | Service down or config error | `Ensure `POSTGRES_PASSWORD` in `.env` matches the one used during first start.` |

| Container fails to start | Port conflict | Check `docker ps` for used ports or use `nikame stop` |
